Use this page when you already have an app, agent, or API route and want to try cascadeflow quickly.

## Fastest Options

| Option | Best when | Why it is fast |
|---|---|---|
| Vercel AI SDK drop-in | You already use `useChat` or AI SDK handlers | Minimal wiring, keeps the existing frontend flow |
| Keep your SDK via gateway or proxy | You already use OpenAI- or Anthropic-style clients | Change the base URL instead of rewriting the app |
| Use cascadeflow directly | You can change the runtime call site | Full control and the clearest long-term integration path |

## Option 1: Vercel AI SDK Drop-In

Use [/integrations/vercel-ai](/integrations/vercel-ai) when you already have a Next.js or Node app using the Vercel AI SDK.

Important example:
- [examples/vercel-ai-nextjs](https://github.com/lemony-ai/cascadeflow/tree/main/examples/vercel-ai-nextjs)

## Option 2: Keep Your Existing SDK

Use a gateway or proxy-style integration when the priority is minimal app change.

Important docs:
- [gateway.md](https://github.com/lemony-ai/cascadeflow/blob/main/docs/guides/gateway.md)
- [proxy.md](https://github.com/lemony-ai/cascadeflow/blob/main/docs/guides/proxy.md)

Important examples:
- [examples/gateway_client_openai.py](https://github.com/lemony-ai/cascadeflow/blob/main/examples/gateway_client_openai.py)
- [examples/gateway_client_anthropic.py](https://github.com/lemony-ai/cascadeflow/blob/main/examples/gateway_client_anthropic.py)

## Option 3: Direct Library Integration

Use the direct library path when you want the cleanest runtime-intelligence setup and can update the call site.

Start here:
- [/get-started/quickstart](/get-started/quickstart)
- [/developers/overview](/developers/overview)

## Guidance

- Choose the path that reduces adoption friction first.
- Start in `observe` even when integrating fast.
- Move to deeper runtime governance once the initial value is proven.
