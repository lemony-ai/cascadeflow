Use this section when you need to understand what cascadeflow is, why the in-process approach matters, and how to get the first rollout live quickly.

## Why Teams Start Here

| Advantage | Why it matters |
|---|---|
| Inside-the-loop control | Runtime decisions happen where agent cost, delay, and failure actually happen |
| No proxy latency tax | In-process control avoids roughly 40-60ms of extra overhead per model or tool call |
| Multi-dimensional optimization | Cost, latency, quality, budget, compliance, and energy can be managed together |
| Runtime governance | KPI and policy intent can shape behavior while the agent is running |
| Transparent rollout | `observe` mode gives visibility before enforcement |
| Measurable value | Summaries and traces show impact on real workflows |

## Start In Order

1. Read the product framing in [/get-started/introduction](/get-started/introduction).
2. Follow the rollout path in [/get-started/quickstart](/get-started/quickstart).

4. Install the right package in [/get-started/installation](/get-started/installation).
5. Review the two-engine model in [/get-started/how-it-works](/get-started/how-it-works).
