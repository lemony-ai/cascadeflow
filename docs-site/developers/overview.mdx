# For Developers

Use this section when you are wiring cascadeflow into an application, framework, or agent runtime.

## Core Concepts You Need Once

- `init(mode=...)` activates the harness globally.
- `run(...)` scopes a budgeted, traceable execution.
- `@agent(...)` attaches policy metadata to an agent function.
- Integrations wrap the framework extension point instead of changing your agent logic.

## Recommended Build Order

1. Enable `observe` mode and inspect summaries plus traces.
2. Add a single framework integration or direct API usage path.
3. Introduce budgets or compliance after you have real runtime data.
4. Expand into KPI weighting, tool gating, and example-based patterns only where needed.
