# cascadeflow

The in-process intelligence layer for AI agents. Optimize cost, latency, quality, budget, compliance, and energy inside the execution loop, not at the HTTP boundary.

Use cascadeflow when you need runtime control, not just model routing. It is a decision system inside the agent loop: every model call, tool call, and handoff can be measured, scored, and steered in real time.

## Why This Direction Matters

| Strategic point | What it means in practice |
|---|---|
| Inside-the-loop control | Decisions happen at agent decision boundaries, not just HTTP request boundaries |
| Multi-dimensional optimization | Cost, latency, quality, budget, compliance, and energy are managed together |
| Business logic injection | KPI and policy intent can shape runtime behavior directly |
| Immediate actionability | The system can `allow`, `switch_model`, `deny_tool`, or `stop` |
| Transparent governance | Every decision is traceable, attributable, and auditable |
| In-process latency advantage | No proxy tax on every call, which matters as multi-step agent loops compound |

## Core Workflow

<CodeGroup>

```python Observe (zero-change)
import cascadeflow
cascadeflow.init(mode="observe")
# All OpenAI/Anthropic SDK calls are now tracked.
```

```python Scoped Run
import cascadeflow
cascadeflow.init(mode="enforce")

with cascadeflow.run(budget=0.50) as session:
    result = await agent.run("Analyze this dataset")
    print(session.summary())
```

```python Decorated Agent
import cascadeflow
cascadeflow.init(mode="enforce")

@cascadeflow.agent(budget=0.20, compliance="gdpr")
async def my_agent(query: str):
    return await llm.complete(query)
```

</CodeGroup>

## What Gets Managed

| Dimension | What it controls | Example |
|---|---|---|
| **Cost** | USD per LLM call from pricing table | Budget cap of $0.50 per run |
| **Latency** | Wall-clock milliseconds per call | Max 2000ms per call |
| **Quality** | Model quality priors for routing | 60% weight on quality KPI |
| **Budget** | Cumulative spend tracking and caps | Per-user daily limits |
| **Compliance** | Model allowlists per regulation | GDPR: only gpt-4o, gpt-4o-mini |
| **Energy** | Compute-intensity coefficients | Carbon-aware model selection |
