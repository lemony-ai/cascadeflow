This page is the shortest reliable map for external agents that need to modify or integrate cascadeflow without reading the whole repo.

## Canonical Product Facts

- cascadeflow is an in-process intelligence layer for AI agent workflows.
- The primary harness modes are `off`, `observe`, and `enforce`.
- The main control surface is `init`, `run`, `@agent`, session summaries, and session traces.
- The key runtime dimensions are cost, latency, quality, budget, compliance, and energy.
- The main actions are `allow`, `switch_model`, `deny_tool`, and `stop`.
