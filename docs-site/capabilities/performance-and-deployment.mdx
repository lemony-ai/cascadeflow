The in-process model is a structural advantage. Proxy-based optimization typically adds around 40-60ms per model or tool call. In deeper agent loops that compounds into avoidable task latency. cascadeflow avoids that proxy tax by staying inside the runtime.

## Why This Matters

| Concern | Why cascadeflow helps |
|---|---|
| Real-time UX | lower added latency per step |
| Throughput | less compounding delay across multi-step workflows |
| SLA performance | more predictable runtime overhead |
| Deployment flexibility | works across direct SDKs, frameworks, local models, and edge-style setups |
