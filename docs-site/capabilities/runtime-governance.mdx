Runtime governance is where cascadeflow moves beyond routing and becomes a decision system inside the agent loop.

## What Runtime Governance Means

| Concern | Runtime control |
|---|---|
| Spend | run budgets, tenant budgets, cost-aware routing |
| Risk | compliance-aware model allowlists and runtime stop conditions |
| Reliability | traceable decisions and bounded tool loops |
| Business intent | KPI weighting across quality, cost, latency, and energy |
| Operational control | `allow`, `switch_model`, `deny_tool`, `stop` |
